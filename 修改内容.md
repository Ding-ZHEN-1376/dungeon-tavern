# 修改内容

## 文件修改

- `new_demo/scripts/interactable.gd`：在原有交互逻辑上新增菜单打开配置，使可交互对象能通过 `menu_id` 调用菜单系统。
- `new_demo/scripts/ui/MenuManager.gd`：负责根据 `menu_id` 创建、关闭和切换对应的菜单 UI。
- `new_demo/scenes/ui/MenuManager.tscn`：作为 `MenuManager` 的 Autoload 场景入口。
- `new_demo/scripts/ui/menus/BaseInteractionMenu.gd`：定义菜单通用信号、初始化接口和关闭逻辑, 作为菜单的基类.
- `new_demo/scripts/ui/menus/': 各类菜单的逻辑功能承载脚本.
- `new_demo/scenes/ui/menus/brewing/BrewingMenu.tscn`：提供酿酒桶菜单的基础 UI 画面。

## 2. UI系统逻辑 (必读)

1. 玩家靠近酿酒桶并按下交互键后,  `interactable.gd` 根调用菜单UI管理系统 MenuManager.
2. MenuManager 根据菜单 ID 创建对应的菜单 UI. 
3. 菜单 UI 根据用户点击的按钮, 发出信号.
4. 信号目前未关联到任何实际逻辑系统 (等酿酒系统实现后, 再将信号关联到酿酒的逻辑)


## 3. 实现的理由

1. 游戏场景中需要开启不同种类的菜单, 需要一个菜单 manager 来管理.
2. 不同种类菜单有相同的基本功能, 通过一个基类来实现基本功能.
3. 通过制作不同的菜单 .tsnc 文件, 可以方便实现不同的功能和视觉效果.
