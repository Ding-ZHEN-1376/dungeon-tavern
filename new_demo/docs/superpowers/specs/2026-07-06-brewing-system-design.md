# 酿酒系统，剧情交互系统与战斗系统

## 目标

实现这样的总体效果：玩家扮演一个地牢酒馆的老板，通过收集各种酿酒素材酿造不同的酒。每天傍晚酒馆开门后，就会有NPC客人来和玩家进行剧情交互，交互的内容基于玩家给NPC提供的酒。当NPC通过喝酒满足一定条件后，玩家就可以接收到NPC的任务，包括但不限于战斗任务，酿造任务等。在一系列任务完成后，根据完成的情况达到结局。

但我们先做酿酒系统。目标是做出一个可以反复游玩的闭环：

0. 设置一个初始的NPC，不仅是引导酿酒，同时也推动简单的剧情。
1. 玩家获得材料（在demo里可以直接交互采集节点获得足够的道具）。
2. 玩家选择材料、工艺、木桶和封装时间开始酿造。
3. 时间推进后酒完成。
4. 傍晚 NPC 来店里买酒。
5. 玩家选择NPC所需要的酒出售，这里可以通过推断NPC的话推断NPC喜欢的酒是什么样子的。
6. NPC 根据满意度付钱、离店、停留或触发事件。NPC对酒的评价会被记录，从而可以指导下一次酿酒的策略。

## 核心设计原则

### 品质、风味、满意度分离

酿酒系统需要拆成三层判断：

| 层级 | 作用 | 示例 |
|---|---|---|
| 酒的品质 | 这瓶酒做得好不好 | 普通、稀有、史诗、传说 |
| 酒的风味 | 这瓶酒是什么类型 | 光灵甜酒、暗咒烈酒、相魂梦酒 |
| NPC 满意度 | 这瓶酒适不适合这个人 | 猎人满意，贵族讨厌 |

例如，一瓶“传说品质的暗咒烈酒”工艺完成度很高，但如果卖给想喝甜酒的普通旅人，旅人可能仍然不满意。反过来，一瓶“普通品质的光灵甜酒”虽然品质不高，但如果刚好符合疲惫旅人的需求，也可以得到不错评价。

### 玩家应该能推理，而不是猜答案

NPC 不应该直接说“我要光 8、灵 4 的酒”。他们应该用自然语言表达需求：

```text
我今天从地牢回来，想喝点能安神的酒，别太冲。
```

这句话背后可以对应：

```text
目标属性：灵、魂
讨厌属性：咒
最低品质：普通
预算：中等
```

玩家通过 NPC 说法、过往日志、材料信息和尝试结果逐渐理解系统，当然也可以通过收集道具等方法得到更多提示，例如购买到了《地下生物图鉴》，从而了解到部分生物对某些材料的喜好程度，或者直接买到了某种酒的配方和制作工艺，等等。

但傍晚开业时间并非只有NPC来喝酒，也有其他的角色来喝酒，只是你可能没法和他们对话。所以在开业前应当设计1到3种酒作为主打酒水清单。到了营业结束时，给出一个收银清单。例如：从顾客处收到费用/材料/等等。

### 大多数酒都能卖，但不是都适合卖

不建议做大量硬性限制。普通酒、怪酒、失败酒大多可以卖给某些 NPC，只是效果不同。真正不能卖或会被拒绝的情况应当少量存在：

- 酒还没有酿好。
- 酿出的酒存在危险效果。
- NPC 有最低品质要求，但酒低于要求。
- 酒带有违禁属性，例如咒、魂、暗过高，需要特殊顾客或许可证。


## 酿酒流程

一次酿酒由四个阶段组成：

1. 添加素材。
2. 选择酿造工艺。
3. 选择封装方式和等待时间。
4. 酿造完成并生成成品酒。

当前项目中，玩家靠近酿酒节点按 `E` 打开酿酒菜单。第一版可以在一个菜单流程中完成前三步，然后把结果记录为“封装中的酒”。玩家不需要每天重复操作同一桶酒。

### 添加素材

配方结构：

```text
水 + 酿酒作物 + 风味材料 + 特殊材料
```

四类材料的定位不同：

| 类型 | 作用 | 示例 |
|---|---|---|
| 水 | 决定酒的底盘和相性 | 清泉水、地牢黑水、梦露水 |
| 酿酒作物 | 决定基础酒体 | 灵能蘑菇、月麦、苦根 |
| 风味材料 | 决定主风味方向 | 月光花、火椒、幽影莓 |
| 特殊材料 | 改变反应、稳定度或事件倾向 | 封魂盐、雾核、古树脂 |

材料不只是物品名和稀有度，还应该有六维属性。

## 六维属性系统

六维属性用于描述材料和酒的风味画像（举例，可以简化）：

| 属性 | 风味含义 | 玩法含义 |
|---|---|---|
| 光 | 清澈、甜、净化、明亮 | 适合旅人、贵族、治疗类需求 |
| 灵 | 草本、菌香、轻盈、安神 | 适合疲惫者、学者、温和需求 |
| 相 | 幻觉、变化、气态、梦境入口 | 适合雾化酒、神秘顾客、实验酒 |
| 暗 | 厚重、苦、地牢感、深沉 | 适合猎人、地牢居民、黑市顾客 |
| 咒 | 辛辣、刺激、危险、爆发力 | 适合竞技场、猎人、冒险者 |
| 魂 | 余韵、记忆、陈年、仪式感 | 适合故事事件、长熟成酒、特殊任务 |

### 材料示例

```text
普通灵能蘑菇（普通的全属性之和等于5）：
光 0，灵 5，相 0，暗 0，咒 0，魂 0

稀有灵能蘑菇（稀有的全属性之和等于10）：
光 0，灵 10，相 0，暗 0，咒 0，魂 0

史诗月光花（史诗的全属性之和等于15）：
光 7，灵 3，相 4，暗 0，咒 0，魂 1

传说火椒（传说的全属性之和等于20）：
光 0，灵 0，相 0，暗 2，咒 10，魂 8
```

材料品质影响属性上限。普通材料可以做出稳定的基础酒，稀有材料更容易做出极端、高级或事件型风味。

### 水和作物的底盘作用

水和作物不只是基础材料，它们会影响后续属性反应。

示例：

```text
清泉水：
光、灵效果 +15%
咒效果 -20%
稳定度 +10

地牢黑水：
暗、咒效果 +20%
光效果 -30%
浑浊缺陷概率 +15%

梦露水：
相、魂效果 +20%
最佳熟成时间更严格
```

如果玩家用“地牢黑水 + 月光花”，光属性会被压制，并且可能出现“浑浊”缺陷。用“清泉水 + 月光花”则更容易得到清澈、明亮的甜酒。

## 属性反应

第一版不需要完整的复杂反应表，只做少量明显、可记忆的反应。

| 组合 | 反应名 | 效果 |
|---|---|---|
| 光 + 灵 | 清辉 | 提升清澈、安神评价，适合普通顾客 |
| 暗 + 咒 | 深渊刺激 | 适合猎人和冒险者，普通顾客可能害怕 |
| 相 + 魂 | 梦境余韵 | 容易触发神秘事件或特殊对话 |
| 光 + 暗 | 浑浊冲突 | 降低稳定度，可能生成浑浊缺陷 |
| 灵 + 咒 | 刺鼻冲突 | 可能生成刺鼻缺陷 |
| 咒 + 魂 | 失控 | 强力但危险，适合后期稀有酒 |

示例：

```text
清泉水 + 灵能蘑菇 + 月光花 + 古树脂
主属性：光、灵
反应：清辉
结果倾向：清澈、安神、甜润，适合旅人和贵族
```

```text
地牢黑水 + 苦根 + 火椒 + 封魂盐
主属性：暗、咒
反应：深渊刺激
结果倾向：厚重、辛辣、危险，适合猎人和竞技场顾客
```

## 品质算法

品质表示“酒做得好不好”，不直接表示“酒是否适合某个 NPC”。品质主要由制作过程决定。

建议公式：

```text
品质分 =
基础分
+ 材料品质分
+ 工艺匹配分
+ 酿造时间分
+ 木桶加分
+ 少量随机波动
- 缺陷惩罚
```

品质等级：

| 分数 | 等级 |
|---:|---|
| 0-49 | 普通 |
| 50-74 | 稀有 |
| 75-89 | 史诗 |
| 90-100 | 传说 |

示例：

```text
基础分：40
材料品质分：12
工艺匹配分：10
酿造时间分：8
木桶加分：0
随机波动：+3
缺陷惩罚：-0
最终品质分：73
品质：稀有
```

这瓶酒的品质是稀有，但是否卖得好还要看它的风味是否匹配 NPC。

## 工艺系统

第一版建议只开放三种工艺，避免按钮很多但差异不明显。

| 工艺 | 适合属性 | 效果 | 风险 |
|---|---|---|---|
| 发酵 | 光、灵、魂 | 稳定，适合基础酒和长熟成 | 上限较低 |
| 蒸馏 | 暗、咒 | 提升烈度和刺激感 | 容易产生刺鼻或失控 |
| 雾化 | 相、魂、灵 | 适合气态酒、梦境酒 | 熟成时间窗口窄 |

示例：

```text
相魂属性高的材料使用雾化工艺：
更容易得到梦境余韵，适合神秘 NPC。

相魂属性高的材料使用普通发酵：
也能成酒，但特殊风味不明显，品质可能不高。
```

## 封装等待

封装包括木桶材质和等待天数。等待时间不是越长越好，每类酒有最佳区间。

### 木桶示例

| 木桶 | 效果 | 适合酒 |
|---|---|---|
| 普通木桶 | 无额外加分，稳定 | 第一版基础桶 |
| 橡木桶 | 魂 +1，品质 +5 | 长熟成酒、温和酒 |
| 黑木桶 | 暗 +2，咒 +1，稳定度 -5 | 暗咒烈酒 |
| 玻璃密封罐 | 相 +2，避免木味干扰 | 雾化酒、花香酒 |

### 时间示例

```text
光灵甜酒：
最佳 2-3 天
少于 2 天：生涩
超过 4 天：香气衰退

暗咒烈酒：
最佳 4-6 天
少于 4 天：刺激不足
超过 7 天：过桶、苦涩

相魂梦酒：
最佳 3-4 天
少于 3 天：梦境感不足
超过 5 天：不稳定，可能失控
```

## 缺陷系统

缺陷不是单纯失败，而是给玩家反馈，让玩家知道哪里出了问题。

| 缺陷 | 可能原因 | 影响 |
|---|---|---|
| 生涩 | 熟成时间太短 | 品质下降，普通 NPC 不喜欢 |
| 过熟 | 熟成时间太长 | 魂属性可能增加，但清爽感下降 |
| 浑浊 | 光暗冲突、黑水压制光属性 | 贵族和普通旅人讨厌 |
| 刺鼻 | 灵咒冲突、蒸馏失败 | 猎人可能接受，旅人讨厌 |
| 失控 | 咒魂过高、稳定度过低 | 高风险，可能触发特殊事件 |

示例：

```text
一瓶“刺鼻的暗咒烈酒”对普通旅人是差酒，但对刚从地牢回来的猎人可能仍然有吸引力。
```

## NPC 售卖系统

傍晚开业后，NPC 逐渐进入酒馆。玩家和 NPC 对话，NPC 提出需求，玩家从成品酒里选择一瓶出售。

### 酒批次与售卖份数

酿造完成后不建议得到“只能卖给一个客人的一瓶酒”。如果一桶酒只能服务一个 NPC，等待 2 天或更久的酿造成本会过高，玩家会频繁遇到“酒还没好、傍晚没东西可卖”的空转问题。

推荐规则：

```text
一次酿造 = 一个酒批次
一个酒批次 = 多份可售酒
普通 NPC 买酒 = 消耗 1 份
追加订单或特殊委托 = 消耗多份
```

示例：

```text
清辉蘑菇酒
品质：稀有
属性：光 8，灵 12，相 1，暗 0，咒 0，魂 2
反应：清辉
缺陷：无
剩余份数：4
```

当疲惫旅人购买一杯后：

```text
售出：清辉蘑菇酒 1 份
剩余份数：3
满意度：88
结果：多付钱，并留下材料线索
```

第一版建议的批次数量：

| 封装规格 | 份数 | 设计用途 |
|---|---:|---|
| 小桶 | 3 | 低成本实验，适合稀有材料试做 |
| 普通桶 | 4 | 第一版默认规格 |
| 大桶 | 6 | 后续扩展，适合稳定量产 |
| 珍贵桶 | 4 | 份数不增加，但提升品质或风味 |

这个设计让一桶酒变成经营库存，而不是一次性奖励。玩家需要思考：

```text
这份稀有光灵酒要不要现在卖给普通旅人？
要不要留给可能出现的贵族客人？
今晚没人要暗咒烈酒，要不要存到猎人来店时再卖？
```

为了避免一桶酒支撑太久，第一版先用固定份数限制即可。后续可以加入最佳饮用期、顾客不接受重复酒、声望越高需求越挑剔等规则。

### 售卖流程

第一版售卖流程建议如下：

1. NPC 进店并提出需求。
2. 玩家打开可售酒列表。
3. 每个酒批次显示名称、品质、主风味、缺陷和剩余份数。
4. 玩家选择一个酒批次售出 1 份。
5. 系统计算满意度和收入。
6. 被选中的酒批次剩余份数减 1。
7. NPC 根据满意度离店、停留、追加订单或触发事件。

追加订单示例：

```text
地牢猎人喝到满意的暗咒烈酒后：
“这酒不错，给我再装两份，我明天进地牢要带上。”

额外消耗：暗咒烈酒 2 份
奖励：金币 + 地牢材料
```

这个流程保留了 NPC 对话选择的经营感，同时避免每个顾客都需要重新酿一桶酒。

### NPC 需求结构

每个 NPC 至少包含：

```text
显性需求：NPC 说出口的需求
目标属性：希望酒具备的风味
讨厌属性：不喜欢的风味
最低品质：拒绝过低品质的酒
预算：决定最高支付能力
耐心：决定是否愿意等待或继续对话
停留概率：满意后是否留在店内触发事件
```

### NPC 示例

```text
疲惫旅人
台词：我今天走了一整天，想喝点温和、能安神的酒。
目标属性：灵、魂
讨厌属性：咒
最低品质：普通
预算：低
满意后：可能告诉玩家某种蘑菇的采集地点
```

```text
地牢猎人
台词：给我来点够劲的，最好能压住地牢里的寒气。
目标属性：暗、咒
讨厌属性：光过高
最低品质：普通
预算：中
满意后：可能送战斗采集材料
```

```text
贵族客人
台词：我要招待客人，酒必须清澈、漂亮，不能有奇怪气味。
目标属性：光、灵
讨厌属性：暗、咒、浑浊、刺鼻
最低品质：稀有
预算：高
满意后：提升酒馆声望
```

```text
神秘术士
台词：我想找一种能让我梦见旧日回声的酒。
目标属性：相、魂
讨厌属性：无明确讨厌，但讨厌过低稳定度
最低品质：稀有
预算：高
满意后：触发特殊事件或解锁材料情报
```

### 满意度算法

满意度由酒和 NPC 的匹配程度决定：

```text
满意度 =
需求匹配分
+ 个人偏好分
+ 酒品质加成
+ 稀有度/新奇度加成
- 禁忌属性惩罚
- 缺陷惩罚
- 价格过高惩罚
```

满意度结果：

| 满意度 | 结果 |
|---:|---|
| 0-30 | 生气，少付钱或降低声望 |
| 31-60 | 勉强接受，低价成交 |
| 61-80 | 满意，正常高价 |
| 81-100 | 非常满意，多给钱，可能停留 |
| 100+ | 惊喜，触发事件、给线索或赠送材料 |

示例：

```text
NPC：疲惫旅人
需求：灵、魂，讨厌咒
酒：稀有品质的光灵甜酒，缺陷：无
结果：灵属性匹配，咒属性低，品质加成不错
满意度：高
可能结果：多付钱，并告诉玩家“月光花适合和清泉水一起用”
```

```text
NPC：贵族客人
需求：光、灵，讨厌暗、咒、浑浊
酒：史诗品质的暗咒烈酒，缺陷：刺鼻
结果：品质很高，但风味完全不匹配
满意度：低
可能结果：拒绝或低价成交，酒馆声望下降
```

## NPC 停留与事件

NPC 满意后不一定立刻离开。部分 NPC 会留在酒馆，玩家可以继续对话。

停留事件可以提供：

- 材料情报：某种材料和某种水相性好。
- 地牢线索：某天某区域更容易获得特殊材料。
- 交易机会：用酒交换稀有风味材料。
- 剧情事件：神秘术士请求特定相魂酒。
- 经营反馈：顾客开始传播某类酒的名声。

示例：

```text
猎人喝到满意的暗咒烈酒后没有离开。
再次对话：
“你这酒有点像黑水洞深处的味道。下次你去那里，记得带点封魂盐，能压住那股失控劲。”

奖励：
获得封魂盐 x1，或解锁“黑水 + 封魂盐可以降低失控概率”的情报。
```

## 第一版设计闭环

第一版应该聚焦最小可玩闭环，不做过多内容。

### 第一版内容范围

材料：

- 水：清泉水、地牢黑水、梦露水
- 作物：灵能蘑菇、苦根
- 风味材料：月光花、火椒、幽影莓
- 特殊材料：古树脂、封魂盐、雾核

工艺：

- 发酵
- 蒸馏
- 雾化

封装：

- 普通木桶
- 橡木桶
- 黑木桶

NPC：

- 疲惫旅人
- 地牢猎人
- 贵族客人
- 神秘术士

营业规则：

- 每晚来 2-3 个 NPC。
- 每个 NPC 提供一句需求台词。
- 玩家从已完成的酒批次中选择一种酒，售出 1 份。
- 系统计算满意度、收入和是否触发事件。

### 第一版完整游玩例子

第 1 天白天，玩家通过探索获得：

```text
清泉水 x1
灵能蘑菇 x1
月光花 x1
古树脂 x1
```

玩家在酿酒节点开始酿造：

```text
水：清泉水
作物：灵能蘑菇
风味材料：月光花
特殊材料：古树脂
工艺：发酵
木桶：普通木桶
等待：2 天
```

系统预测或记录：

```text
主属性：光、灵
可能反应：清辉
预计完成：第 3 天
```

第 3 天酒完成：

```text
名称：清辉蘑菇酒
品质：稀有
属性：光 8，灵 12，相 1，暗 0，咒 0，魂 2
反应：清辉
缺陷：无
剩余份数：4
```

第 3 天傍晚，疲惫旅人来店里：

```text
台词：我今天走了一整天，想喝点温和、能安神的酒。
```

玩家出售“清辉蘑菇酒”。系统计算：

```text
灵属性匹配：高
魂属性略微匹配：低
咒属性惩罚：无
品质加成：稀有
缺陷惩罚：无
满意度：88
结果：非常满意，多给钱，并停留对话
剩余份数：3
```

停留对话奖励：

```text
旅人告诉玩家：月光花和清泉水很搭，但不要用地牢黑水压它。
```

这个闭环同时验证了采集、酿造、等待、完成、售卖、满意度和情报反馈。

## Godot 实现可行性

当前项目已经具备适合接入酿酒系统的基础：

- `MenuManager` 可以打开酿酒菜单。
- `BrewingMenu` 已有酿酒 UI 入口雏形。
- `MaterialInventory` 可以管理材料和成品酒。
- `TimeSystem` 可以推动天数和时段。
- `ActivitySystem` 可以承载采集、交易、酿酒、营业等活动。

建议后续新增系统：

| 系统 | 责任 |
|---|---|
| `BrewingSystem` | 管理酿造中酒、完成检查、品质和风味计算 |
| `BrewingDatabase` | 定义水、作物、风味材料、特殊材料、工艺、木桶 |
| `CustomerSystem` | 生成傍晚 NPC、需求、满意度和奖励 |
| `WineDatabase` 或成品酒数据结构 | 保存每瓶酒的品质、六维属性、缺陷和来源 |
| `BrewingLog` | 记录玩家实验结果和材料情报 |

第一版可以全部使用字典和常量表实现，暂时不需要复杂资源编辑器。等规则稳定后，再考虑迁移到 Godot `Resource` 或外部表格。

## 后续扩展方向

第一版跑通后，可以逐步扩展：

- 更复杂的 NPC 个人关系和回头客。
- 酒馆声望影响高预算顾客出现概率。
- 特定 NPC 委托指定风味酒。
- 材料图鉴记录已发现相性。
- 酒名根据主属性、工艺、缺陷自动生成。
- 危险酒触发事故、黑市或剧情事件。
- 高级器具解锁更强工艺，例如精密蒸馏器、月雾器、魂封坛。

## 总结

酿酒系统的核心不是单纯堆稀有材料，而是让玩家理解并利用三件事：

```text
品质：酒做得好不好。
风味：酒是什么性格。
满意度：这瓶酒适不适合这个 NPC。
```

第一版只要把这个闭环跑通，后续就可以自然扩展材料、工艺、NPC、事件和剧情，而不会推翻已有设计。
